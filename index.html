居易工作心得

　　在居易待了一年多的時間，最後因為個人生涯規劃而離開了公司，雖然待的時間不長，但是第一份工作的經驗讓我學到了許多，不單是工作技能上，待人處事上也是，在學校只要想著如果把事情完成就好，而工作之後了解到，程式的效率及環境的限制也是很重要的部分，再加上與PM、PQC或客戶溝通等，都讓我學到很多。
　　工作主要是嵌入式系統開發，總共接觸了三個開發的專案，分別為：GPON Router、AP Management、PoE Switch，大部分是在各種平台上開發所需要的process；因為是新部門的關係，而且人數很少，所以我們從底層到前端都需要自己做，每個人各自負責不同的功能，從driver的銜接到網頁設計或是CLI的格式，都要自己做，也因此學到了各種技術。
　　到了第三個專案，我負責建置整個網頁的架構，主要是沿用前個專案的架構搬到Switch，再根據Switch的需求及硬體限制做更改。

Development of integrated graphic user interface based on MATLAB for proton MR spectroscopy analysis with LCModel

柯正雯 教授（中山大學）

MATLAB、Linux C、Java
內容簡介：LCModel為核磁共振頻譜十分具有公信力的分析軟體，但是價錢十分昂貴且綁定硬體，這兩個特點使的我們需要使用時只能到特定的電腦上使用，這也促成我研究這個主題的動機。目的就是使用MATLAB來設計一套能夠與LCModel溝通的系統並且加上了其他方便分析的功能。MATLAB在分析與計算上能夠提供很多幫助，又兼具跨平台的優點，所以選擇使用MATLAB開發。溝通方式為SSH及FTP，LCModel為Linux的程式。系統結構採用MrStruct的理念設計，使用物件導向的方式設計整個系統。

大學專題研究

智慧型就診隨身提醒設備

馬詠程 教授（長庚大學）

C、HTML、PHP、Unix
合作組員：林曉楓、蔡宗佑、曾恕勃
內容簡介：為了加強病患在醫院等待時的照護，並讓病患能夠活用等待時間，我們設計這套系統。當病患掛號時領取我們的發送器，在我們佈下接收器的環境中活動時，隨時回傳位置資訊，若病患遇到緊急狀況也可以透過發送器向院方求救，護士可以透過網頁觀察病患所在位置，位置資訊將存進分散式資料庫，而未來可以對其加以利用。
伺服器端(Linux)：因為接收器的數量多，我們使用分散式資料庫方式設計，不同的接收器透過我們的Hash table儲存進入不同的多個資料庫。儲存方式為Link list，藉以加快資訊儲存速度。傳輸方式為TCP，保障傳輸可靠性。使用者介面為網頁，方便使用者直接對病患資訊做更動。
發送器端：設計多種不同省電的睡眠方式。根據病人不同的狀態，使用不同的省電方式，藉以達到最高的省電效益。
學習歷程：完成這項專題讓我學會如何在多種程式語言及多種平台間溝通，以及實做了上課時學到的Hash和Link list的Database。不過最重要的還是學會了與組員間的溝通，讓計畫能夠順利的實現。

實習成果（１／２）

PowerRMS

C#、ASP.NET、MS SQL
內容簡介：Power RMS為監視與控制終端設備的系統軟體，包含了RMS-Client、RMS-Server、RMS-Management Tool三部分。
Client安裝於終端設備上，其主要功能有兩部分：其一是監控終端設備執行時的設備狀態，並且將狀態資訊記錄為log，並通過Web Service Interface上傳至Server。其二是直接接受Server Management Tool通過網路命令，執行開關機等。
Server安裝於伺服器(Windows XP + IIS)上，其主要功能是接收Client上傳的Log資訊，並保存於系統資料庫中。
Server Management Tool安裝於伺服器主機或者PC(需開啟SQL TCP/IP Connection)上，其主要功能是為管理人員提供圖形化的操作介面進行管理。
